07 — Brand

Voice & Tone

Open Window speaks as an inspiring creative mentor — confident in industry expertise, approachable to prospective students, and precise when it matters. Voice stays consistent; tone adapts to context.

Brand Positioning Statement

Open Window is bold without being arrogant, creative without being vague, and professional without being corporate. We speak to future makers — students, parents, industry partners, and alumni — with clarity and energy.

Voice Attributes

Inspiring

We paint possibility — careers in film, design, animation, and creative tech.

"Build the portfolio that gets you hired." "Qualifications are available in various disciplines."

Direct

We respect the reader's time. Say what matters, then offer the next step.

"Apply by 30 November. 12 places left." "Applications are encouraged ahead of the closing date."

Industry-Led

We ground claims in real creative industry outcomes and accreditation.

"CHE-accredited. Industry briefs from day one." "We strive to hopefully meet industry standards."

Human

We use "you" and "we". We celebrate student work authentically.

"Your reel is ready — share it with our community." "The learner's portfolio submission has been received."

12 — Experience

Accessibility Standards

Open Window digital properties must meet WCAG 2.2 Level AA as a minimum. Accessibility is a design requirement — not a post-launch audit item. These standards apply to openwindow.co.za, the community platform, and all WordPress/Elementor builds.

WCAG 2.2 — Four Principles
👁️
Perceivable

Text meets contrast minimums. Images have meaningful alt text. Video has captions. Information is never conveyed by colour alone. Content reflows at 200% zoom without horizontal scroll.

⌨️
Operable

All functionality works via keyboard. Focus is always visible and not obscured by sticky UI. No keyboard traps. Touch targets ≥ 44×44px. Users can pause, stop, or hide moving content.

📖
Understandable

Clear labels on every input. Errors identified in plain language with recovery steps. Consistent navigation across pages. lang attribute set on the document element.

🤝
Robust

Semantic HTML first; ARIA only when native elements are insufficient. Valid name, role, and value on all custom components. Compatible with NVDA, JAWS, and VoiceOver.

Key WCAG 2.2 AA Criteria
Principle Criterion Requirement
Perceivable 1.1.1 Non-text Content Descriptive alt on images; decorative images use alt=""
1.4.3 Contrast (Minimum) 4.5:1 normal text · 3:1 large text (18pt+ or 14pt bold)
1.4.11 Non-text Contrast UI components and focus indicators: 3:1 against adjacent colours
Operable 2.1.1 Keyboard All functionality available via keyboard; no traps
2.4.11 Focus Not Obscured (Min.) Focused element not fully hidden by sticky headers or overlays
2.5.8 Target Size (Minimum) Interactive targets ≥ 24×24px (OW standard: 44×44px)
2.2.2 Pause, Stop, Hide Auto-rotating content must be pausable; respect prefers-reduced-motion
Understandable 3.3.1 Error Identification Errors described in text; suggestions provided where possible
3.3.8 Accessible Authentication No cognitive function tests for login; support paste in password fields
Robust 4.1.2 Name, Role, Value Custom widgets expose correct ARIA; form inputs have accessible names

Focus & Interaction Standards

All interactive elements must show a visible focus indicator. Use the brand tech orange ring — never suppress outlines without a custom replacement.

/* OW focus ring — apply to all interactive elements */ :focus-visible { outline: 2px solid var(--ow-color-tech); outline-offset: 2px; } /* Honour reduced motion */ @media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } }
Implementation Checklist
✓ Always implement
  • Logical heading hierarchy (H1 → H2 → H3, no skipping levels)
  • Semantic HTML first; ARIA only when native semantics are insufficient
  • lang="en" (or appropriate language) on the <html> element
  • Skip-to-content link as the first focusable element
  • Keyboard-navigable dropdown menus, modals, and flyouts
  • Visible, persistent labels on all form inputs
  • Minimum 44×44px touch targets on mobile
  • Text resizable to 200% without loss of content or functionality
  • Captions and transcripts for video content
  • prefers-reduced-motion honoured on all animations
✕ Never do
  • Convey meaning by colour alone (add icon + text for states)
  • Auto-play media with sound without user initiation
  • Content that flashes more than 3 times per second
  • Open links in new tabs without warning (rel + visible indicator)
  • Use tabindex values greater than 0
  • Remove native focus outlines with outline: none and no replacement
  • Placeholder text as the only form label
  • Icon-only buttons without aria-label
🔧 Testing protocol: Run automated checks with axe DevTools and Lighthouse on every deploy. Target Lighthouse Accessibility ≥ 95. Supplement with manual keyboard-only navigation and screen reader testing (NVDA on Windows, VoiceOver on macOS/iOS). Test at 200% browser zoom and simulate Slow 3G for South African mobile conditions.
13 — Experience

UX Recommendations

Aspirational best-practice guidance for Open Window digital properties — informed by the design system, education UX standards, and patterns observed across creative-industry institution sites. These recommendations preserve the OW brand identity while reducing cognitive load and improving conversion.

📋 Priorities: High implement first · Medium next sprint · Low roadmap. Category tags indicate the primary area of impact.
01

One primary CTA per page High

Every page should have a single unmistakable primary action — typically "Apply Now" on prospect-facing pages, or "Upload Portfolio" in the community context. Secondary actions use ghost or outline button styles. Competing orange buttons in the utility bar, hero, and inline content create decision fatigue and dilute conversion.

Conversion Visual
02

Simplify navigation to two levels maximum High

The qualifications menu spans three nesting levels (school → level → programme). Consolidate into a two-level mega-menu with scannable columns: Postgraduate · Degree · Higher Certificate · Short Learning Programme. Deep flyouts increase cognitive load, break keyboard navigation, and fail on mobile. See the Navigation section for the target structure.

Navigation Accessibility
03

Keep "Apply Now" persistently visible High

The primary conversion goal is student applications. "Apply Now" should remain visible in the sticky header at all scroll depths — not buried behind dropdown menus. On mobile, consider a persistent bottom bar for the primary CTA. Utility links (Academic Calendar, Community Dashboard) should not compete visually with Apply.

Navigation Conversion
04

Replace auto-rotating heroes with static, purposeful heroes Medium

Carousels that auto-advance fail WCAG 2.2.2 and disorient users scanning for a single message. Use a static hero with one headline, one benefit-led sub-headline, and one CTA. If a carousel is retained, it must have accessible pause controls, keyboard navigation, manual advance only, and respect prefers-reduced-motion.

Accessibility Visual
05

Add a qualification comparison tool Medium

Prospective students frequently compare programmes across schools. A lightweight "compare" feature — select 2–3 qualifications and view duration, NQF level, entry requirements, and career outcomes side-by-side — reduces back-and-forth navigation and supports informed decisions without leaving the site.

Content Conversion
06

Enforce a single typeface system sitewide Medium

The design system specifies Montserrat as the primary web typeface with Avenir Next for brand applications. Multiple font families loaded simultaneously (Poppins, Nunito, Open Sans, etc.) increase page weight, create visual inconsistency, and undermine brand recognition. Limit web UI to Montserrat + Quicksand accent only.

Visual Performance
07

Standardise card patterns across all content lists Medium

News, events, qualifications, and portfolio items should share the card component defined in Cards. Consistent structure (image → meta → title → excerpt → CTA) creates visual predictability — users learn once how to scan content across the entire ecosystem including the community platform.

Visual Content
08

Establish consistent section rhythm on every page Medium

Each major section should follow: eyebrow label → heading → lead paragraph → content → CTA. Apply the spacing scale from Layout — minimum 64px between major sections on desktop, 48px on mobile. Alternate #232323 and #1F1F1F backgrounds to group related content without adding borders.

Content Visual
09

Add progress indicators to multi-step flows Low

Applications, enrolments, and portfolio uploads involve multiple steps. Without a visual stepper, users cannot gauge remaining effort — increasing abandonment. Implement a linear progress indicator using tech orange for the active step, film blue for completed steps, and muted grey for upcoming steps.

Conversion Content
10

Optimise for mobile bandwidth and performance Low

A significant portion of SA traffic is mobile on constrained networks. Serve images in WebP with responsive srcset, lazy-load below-the-fold media, compress hero images under 200KB, and set performance budgets (LCP under 2.5s). Test with Chrome DevTools at Slow 3G. Creative portfolio sites are media-heavy — performance directly impacts bounce rate and accessibility.
